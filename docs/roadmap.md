# Roadmap

## Infrastructure and deployment

Planned infrastructure improvements include:

- improved deployment with HTTPS
- purchase and configuration of a cleaner custom domain
- migration to a more powerful server if production load grows

## Authentication and security

Security-related improvements planned for the project:

- authentication / login service
- better access control
- stronger handling of operational data security
- more structured review of privacy and data protection concerns

## Dog data management

Planned improvements related to dog data:

- photo upload service for dog profiles
- improved media handling
- possible cloud-based data storage strategy for selected assets or operational data

## Seasonal workflow support

The current MVP focuses on a specific operational pattern, but future versions should support more seasonal states.

Planned additions include:

- offseason mode
- autumn training season mode
- broader support for season-dependent planning logic

## Planning model improvements

Team-building and planning logic can be extended beyond the current single-run oriented structure.

Future work may include:

- full-day program layers
- modeling operational programs across an entire workday
- adapting Team Builder to multi-program daily planning
- more advanced balancing logic

## Data quality and validation

A major part of the next phase is not only adding features, but improving trust in the system.

That includes:

- more validation of domain assumptions
- more systematic testing
- expanding test coverage
- refining analytics and planning outputs based on real use

## External dog-performance data

A particularly interesting future direction is integration with collar-based activity data.

This could support:

- more precise workload measurement
- stronger understanding of actual force and effort
- more objective strength estimation for dogs
- better balancing logic in Team Builder
- improved quality of data behind workload-related decisions

## Long-term product direction

The project could evolve from an MVP toward a more robust planning platform by improving:

- infrastructure quality
- data reliability
- operational modeling
- decision support
- reporting and analytics depth

## Guiding principle

The main principle for future work is simple:

keep testing, keep validating, and keep refining the system based on real operational use.
